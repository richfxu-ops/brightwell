Roundelay · milestone 3

The Engine Plan

How we turn the locked design into a running game — in eight small, testable steps, each one provable before the next begins.

The one idea

The engine is a careful bookkeeper

Everything in Roundelay's design is ultimately numbers moving between pots: attention pours from the dawn into the room, from the room onto pieces, spills over ceilings into Standing, seeps back to the town's table. The engine's whole job is to move those numbers around exactly as the locked design says, and never any other way.

So we build it as a pure state machine: one plain data object holds the entire game ("the table"), and the only way anything changes is a function that takes the current state plus one action and returns the next state. No hidden variables, no side effects. Cards stay data — the 49 starter cards are JSON we already have; the engine just knows the fourteen verbs they're written in.

Why this shape matters: it makes the engine equally drivable by a test, a bot, or (later) a UI — all three just feed it actions and read the result. That's exactly what M4's balance simulations need.

The shape

Nine layers, built bottom-up

Each layer only depends on the ones below it — which is also the build order, and (not by accident) the order of the GDD's own layers.

1The table (state)one data object: the run, the board, your deck, hand, room, gleam, purse
2Readsthe "amounts are references" rule — read(room), read(grain:joinery), read(spiral)…
3The fourteen verbsone resolver per primitive, with the firewalls enforced
4The worked morningdawn → draw → chain of plays → wake/spill → dusk settle
5The askingscontracts off the lantern: sizes, fills, staleness, the glad-load payout
6The year27 mornings, 5 legs, the Paling's spiral rings, the combing, the crown
7Growing the deckglad-load bundles, the Round-Fair, courting — how new cards arrive
8The thermometerthe 57 telemetry numbers every run must report
9Bots & the harnesssix archetype players + thousands of seeded runs → hands off to balance
Three ground rules

Decisions to agree on before we start

1 · Same seed, same run — always. All randomness (shuffles, lantern offers, table draws) comes from one seeded random-number generator stored in the state. Run the engine twice with seed 42 and you get the identical game, morning by morning. This is what makes bugs findable ("run seed 8871, morning 14, watch the spill"), tests trustworthy, and balance statistics honest.

2 · The engine narrates everything it does. Every state change also appends a small plain-English-able event to a log: {woke: "the-fired-beam", overkill: 1}. Tests read the log to assert what happened; telemetry (layer 8) is just arithmetic over it; and when we build the playable prototype, the UI simply animates the log. We get replays for free.

3 · A fidelity ladder, not silent shortcuts. Version 1 deliberately simplifies two world-rich systems — the map (full road-memory graph → for now, each dusk offers 2–3 next towns to choose from) and acquisition's outer reaches (full courting web and Fair economy → for now, the glad-load bundle plus a simple Fair pick and the courting terms the Mannerly cards need). Every simplification gets a // FIDELITY: marker and a line in the task file, so nothing quietly contradicts canon — we're deferring detail, never changing the rules.

What we are NOT building in M3: graphics or UI, the full illustrated map, save files, sound, meta-progression. The deliverable is an engine a bot can play a complete, honest run of — that's what M4's balancing needs.
The steps

Eight phases, each one testable

1

The table — state & reads

Small
We buildThe GameState type — calendar, board nodes (with their paling rings and local tables), deck (fired vs held), hand, the room, chain, gleam, handsels, courted stock — plus the read() evaluator that turns any read(...) source into a number from that state.
Why firstEvery other layer speaks in reads. "A card's payoff isn't a fixed number — it reads what earlier cards built" is the game's one clever idea; it has to be rock-solid.
Done whenTests build small states by hand and every read source returns the documented number — including the closed-enum rule (an unknown source is a compile error, not a zero).
2

The fourteen verbs

Medium
We buildOne resolver per primitive — gather, rest, steady, fill, brim, mark-grain, draw, retire, whittle, court, soothe, read, warm, keep — each a pure function: state + effect in, state + events out.
The teethThe five locked firewalls become runtime assertions and tests: brim is the only gleam-writer; season/spiral reads never feed gleam; no verb lowers a waking-mark; gleam gates but is never spent; the Paling touches only the board.
Done whenTable-driven tests cover every verb (including rest's three outcomes: wakes / stays cold / overkills past ceiling), and a "firewall suite" proves each forbidden move is impossible.
3

The worked morning

Medium
We buildThe turn loop: dawn income (base + local-table draw + yesterday's returned surplus), draw to a hand of five, then a chain of plays — each spending room, holding or cooling on stalls, triggering on-play / on-wake / on-chain / on-overkill — and a dusk settle (unspent attention RETURNs to this node's table, freshness ticks, idle handsels dim).
The proofA golden test straight from the GDD: the Kilnfast's documented three-card chain (Setterby Trestle → Calipers → The Fired Beam) played in order wakes the capstone; played out of order, it under-fills — the reach that falls short.
Done whenA scripted sequence of plays runs a full legal morning, and the conservation law holds: every drop of dawn attention is accounted for as SET, overflow, or RETURN — nothing minted, nothing lost.
4

The askings

Medium
We buildContracts from tiers.json: the four tiers (kettle 1 → plea 3 → poem 5 → great 7), lantern offers gated by gleam band, accepting an asking, fill completing it, staleness past its season becoming the spilling (the only way gleam falls), and the glad-load payout that touches every economy at once.
Done whenTests: fulfil pays the documented bundle and resets the node's rings ("rings-in = load-out"); a stale asking costs gleam exactly once; a deliberate flop is EV-negative.
5

The year

Medium
We buildThe run frame: 27 worked mornings across the five legs (4·6·7·7·3), the Paling accreting spiral rings on skipped nodes (3 = plea, 5 = poem, 7 births a great asking + the combing), the simplified travel choice each dusk, the calendar-floored crown (need 10) at the Wintering, and the two endings — stand the crown and win, or gleam-zero into the Quiet Walk.
Done whenA scripted full run plays start to finish; the season multipliers match tiers.json; and the key design assertion holds in a test: a static 7-card apprentice deck cannot stand the crown — growth is forced by the weather.
6

Growing the deck

Medium
We buildAcquisition, at ladder-rung one: the glad-load's taught piece + introduction-jar + bright handsels + map hint; a simple Round-Fair pick when you route through one; courting terms (so the Mannerly's court cards work as printed); and the drains — retire, gifting, freshness lapse — so the deck stays a garden, not a hoard.
Done whenA run's deck actually grows ~7 → ~20 as L6 says; every acquired piece arrives inert (the room still has to wake it); handsels never buy proud stock.
7

The thermometer

Small
We buildThe telemetry module: every run emits the 57 keys in design/round_metrics.json (win/loss, decisions per turn, deck curve, gleam economy, the Untold's pace ratios, the six axis magnitudes…) — computed from the event log, not sprinkled through the engine. Plus the first greedy bot: no strategy, just "make a legal good-ish move," to smoke-test that whole runs complete unattended.
Done when1,000 seeded greedy runs finish without an illegal state, and every one of the 57 keys is populated (no silent nulls).
8

Six temperaments — bots & the harness

Large · split later
We buildOne policy bot per Walking Way (the Kilnfast snowballs, the Fairwright hoards-then-dumps, the Gleaner routes into the grey…), plus the harness: run N×6 seeded games, aggregate the telemetry, and print the balance sheet the GDD's watch-items ask for — crown-stand spread, glad-load power share, dominated pairs, the Untold's tempo ratios.
Done whenThe harness produces its first honest balance report. That report — not this phase — is M4's starting gun; tuning numbers is the next milestone's job (and where we decide whether the game-loop skill runs the loop).

Working rhythm: each phase is its own /task file (scoped → designed → signed off → built on a side branch → reviewed → merged), so you approve each step's design before its code exists — and any phase can pause without leaving the engine broken.
